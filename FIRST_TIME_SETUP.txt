================================================================================
  FIRST TIME SETUP — project
================================================================================

PRE-REQUISITES
--------------
- Docker Desktop installed and running
- Git (to clone the repo)

--------------------------------------------------------------------------------
STEP 1 — Clone the repository (skip if already done)
--------------------------------------------------------------------------------

  git clone <repo-url>
  cd project

--------------------------------------------------------------------------------
STEP 2 — Create the environment file
--------------------------------------------------------------------------------

  copy .env.example .env
  (on Linux/Mac: cp .env.example .env)

  Open .env and set:
    APP_KEY=       <- will be generated in Step 4
    APP_ENV=local
    APP_DEBUG=true

  The database settings are already pre-configured for Docker (see CLAUDE.md).

--------------------------------------------------------------------------------
STEP 3 — Build and start all Docker containers
--------------------------------------------------------------------------------

  docker compose up -d --build

  This starts 4 services:
    - app   -> PHP 8.2 + nginx/php-fpm  -> http://localhost
    - node  -> Node 22 (Vite HMR)       -> http://localhost:5173
    - mysql -> MySQL 8.0                -> localhost:3307
    - redis -> Redis 7                  -> localhost:6379

  The app container entrypoint automatically runs:
    - php artisan migrate --force
    - DB seed (on first start)

  Wait ~30 seconds for all containers to be healthy before proceeding.

--------------------------------------------------------------------------------
STEP 4 — Generate the application key
--------------------------------------------------------------------------------

  docker exec project-app-1 php artisan key:generate

--------------------------------------------------------------------------------
STEP 5 — (Optional) Seed the database manually
--------------------------------------------------------------------------------

  Only needed if the automatic seed did not run or you want a fresh dataset:

  docker exec project-app-1 php artisan db:seed --force

  Full reset (drops everything and re-seeds):

  docker exec project-app-1 php artisan migrate:refresh --seed --force

--------------------------------------------------------------------------------
STEP 6 — Create the storage symlink
--------------------------------------------------------------------------------

  docker exec project-app-1 php artisan storage:link

--------------------------------------------------------------------------------
STEP 7 — Start the frontend dev server (Vite HMR)
--------------------------------------------------------------------------------

  The node container is already running. Frontend is served at:
    http://localhost:5173

  To watch logs:
    docker logs -f project-node-1

--------------------------------------------------------------------------------
DONE — Access the app
--------------------------------------------------------------------------------

  App URL:      http://localhost
  Vite HMR:    http://localhost:5173
  MySQL:        localhost:3307  |  DB: splnet_db  |  User: splnet  |  Pass: splnet
  Redis:        localhost:6379

================================================================================
  USEFUL COMMANDS (day-to-day)
================================================================================

  # Start containers (after first setup)
  docker compose up -d

  # Stop containers (data is preserved)
  docker compose down

  # Stop and DELETE all data (full reset)
  docker compose down -v

  # Rebuild only the app container
  docker compose up -d --build app

  # Run migrations
  docker exec project-app-1 php artisan migrate --force

  # Run tests
  docker exec project-app-1 composer test

  # Open a shell inside the app container
  docker exec -it project-app-1 sh

  # Build frontend for production
  docker exec project-node-1 npm run build

  # View app logs
  docker logs -f project-app-1
